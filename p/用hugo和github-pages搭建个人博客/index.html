用Hugo和Github Pages搭建个人博客

A step by step guid to set up your pesonal blog using Hugo and Github Pages.

Oct 04, 2020

现在随着各种各样的网站生成工具的普及和免费的服务器资源，搭建一个个人网站或博客变得很容易。本文将分享我通过Hugo和Github Pages来搭建这个网站的细节。

Hugo是一个静态网站生成器。有很多现成的主题模板可以用，所以自己无需去设计和实现所有网页渲染的样式。我的这个博客是用的Stack，我喜欢它的圆圆的卡片设计。

1.本地搭建网站

仅仅用Hugo，就可以在本地轻松看到自己网站发布之后的效果。具体步骤如下：

  1. 下载Hugo。我是用Mac，所以直接用homebrew安装Hugo。
    brew install hugo
  1. 用Hugo创建一个网站。这条命令会帮你创建一个名为"testblog"的网站，同时你会在本地文件里看到一个名为“testblog”的文件夹。
    hugo new site testblog
  1. 选择一个主题给这个testblog网站。这里我以Stack主题为例。
    cd testblog

    git clone https://github.com/CaiJimmy/hugo-theme-stack.git --depth=1 themes/hugo-theme-stack

    # 拷贝样例网站的内容至当前目录
    cp -r themes/hugo-theme-stack/exampleSite/content ./    

    # 拷贝默认设置至当前目录
    cp -r themes/hugo-theme-stack/config.toml ./

    # 大功告成，运行如下命令便可在本地查看这个网页了
    hugo server

按照指示，在浏览器中输入http://localhost:1313/浏览。

2.部署上线

网站在本地测试好了之后，就可以通过Github Pages来部署上线了。 Github Pages是Github提供给每个用户的一个免费的网站托管服务和一个对应的用户域名https://<username>.github.io。 Hugo官方网站提供了非常详细的步骤。具体如下

  1. 创建2个git repo，一个用来host网站代码，另一个用来host Hugo生成的静态网页。后者的repo名字需要跟<github_username>.github.io一样。以后每次你发新的博客时，需要把更新push到第二个repo.

  2. git clone <YOUR-PROJECT-URL> && cd <YOUR-PROJECT>

  3. 把在上一步搭建本地网站时的Hugo project拷贝到新的<YOUR-PROJECT> repo. 然后执行 hugo server并查看网页是否在本地可以正常渲染。同样的，也是在 http://localhost:1313访问。

  4. 当你对网站样式满意之后，可以关掉本地服务器，并删除掉当前目录下的public文件夹。每次hugo server执行的时候，都会重新生成网站的内容去覆写这个文件夹。

    rm -rf public
  1. 建一个submodule把<username>.github.io托管的repo指向public目录。这样每次源文件改后，直接执行下一步中的deploy脚本，就可以自动更新网站repo.
    git submodule add -b master https://github.com/<USERNAME>/<USERNAME>.github.io.git public.

  1. 把config.toml中的baseURL改成<USERNAME>.github.io。这么做的目的是网站所有子页面都会以此作为主目录。

  2. 写一个deploy的脚本。每次修改了源码后，直接运行如下命令来获取更新后的网站。

    script/deploy

这样就搞定了。过几分钟就可以直接去https://<username>.github.io访问你的个人网站了！

3.修改样式

拿到一个Hugo主题后，可以按照自己的喜好和需求来修改模板。在推出我的初稿之前，我对Stack theme做了如下修改：

  1. 关掉右边栏的archive和tag cloud
  2. Enable Disqus所以每篇文章后面下面会有留言区
  3. 去掉对文章段落小标题的高亮
  4. 在About页关掉评论区
  5. 去掉了头像右下角的emoji

最终看起来就是你现在正看到的样子。原作者的博客也是基于他的这个模板做的，功能更丰富一些，我觉得挺可爱的。

接下来，我还会持续更新这个博客的样式，也会把值得分享的改动分享给大家。目前的计划包括引入Google Analytics对站点流量进行监控，增加订阅功能和本地搜索功能等。
